Technical Reference

Endpoint handler inside httpserver.cpp. It establishes a short-lived connection context, runs a simple select query, and returns JSON.

GET /api/db-time Handler 
svr.Get("/api/db-time", [](const httplib::Request &, httplib::Response &res) {
  try {
    pqxx::connection c("dbname=postgres user=postgres password=root hostaddr=127.0.0.1 port=5432");
    if (c.is_open()) {
      pqxx::work w(c);
      pqxx::result r = w.exec("SELECT current_timestamp;");
      
      std::string db_time = r[0][0].c_str();
      std::string json_res = R"({"status": "success", "db_time": ")" + db_time + R"("})";
      res.set_content(json_res, "application/json");
    } else {
      res.status = 500;
      res.set_content(R"({"status": "error", "message": "Gagal membuka database"})", "application/json");
    }
  } catch (const std::exception &e) {
    res.status = 500;
    res.set_content(R"({"status": "error", "message": ")" + std::string(e.what()) + R"("})", "application/json");
  }
});

Compiling on Windows and Linux requires linking against ws2_32 (Windows networking) and PostgreSQL dependencies (libpq, libpqxx).

Windows Compilation (g++) 
g++ httpserver.cpp -o httpserver.exe -lws2_32 -lpqxx -lpq
Linux Compilation (g++) 
g++ httpserver.cpp -o httpserver -lpqxx -lpq -pthread
Spin up PostgreSQL with Docker 
docker run --name pg-test -e POSTGRES_PASSWORD=root -p 5432:5432 -d postgres

Common errors when integrating C++ HTTP servers with PostgreSQL databases:

Error: connection to database failed

Verify that the PostgreSQL service is actively running and listening on port 5432. Double-check your host address IP (e.g. 127.0.0.1) and check firewall credentials.

Error: password authentication failed for user

The password specified in the C++ pqxx connection string does not match the PostgreSQL credentials. Update the connection builder and update the server code accordingly.

Why is my modified index.html not loading?

The server reads index.html into RAM cache on startup. Re-run or restart the C++ executable to load changes: Ctrl+C then restart.

High-Performance Server Architecture

Below are the structural components that power the PgConnect test bench platform.

cpp-httplib Routing

Multi-threaded HTTP router providing simple Get/Post mappings, CORS middleware support, and blazing fast routing latency (sub-millisecond handler processing).

pqxx Connection API

Official C++ driver bindings implementing standard transaction encapsulation (pqxx::work) with strong exceptions handling to prevent thread pool leaks.

Graceful Shutdown

Detects standard OS terminate interrupts (SIGINT/SIGTERM) to clean up ongoing HTTP request handles and close database connections cleanly without segmentation faults.
